## a) Apassi. Asenna Apache 2 käsin. 
> Weppisivun tulee näkyä palvelimen etusivulla. Sivun tulee olla tavallisen käyttäjän muokattavissa, ilman root- tai sudo-oikeuksia.

### Ladataan Apassi!
Minulla on onneksi apassi valmiina, mutta lukioiden iloksi sen saa ladattua: 
```
sudo apt install apache2
```
Vilkaistaanpa, että apache on käynnissä komennolla:
```
sudo systemctl status apache2
```
Jos käynnissä => HYVÄ! 
Sitten vakiosivut pitäisi löytyä "http://localhost".

![vakiosivu](imagesh3/apache2_defaultpage.JPG)
### Käyttöoikeudet
Jotta kaikilla käyttäjillä olisi kivaa, niin laitetaan oikeudet kuntoon.

Aluksi ne näyttävät tältä. Vakiosivu sijaitsee polussa /var/www/html/index.html. Index.html on vakio sivu.

![vakio oikeudet](imagesh3/apache2_defaultopage_access.JPG)

```
-rw-r--r--
```
Nyt muut käyttäjät pääsee vain lukemaan ja tämähän ei tule kuuloonsa. Annetaan kaikille pääsy lukemaan ja ajamaan tiedostoja. Sitten annetaan vakiosivulle sellaiset pääsyoikeudet, että vaikka testi Pena pääsee sinne runoilemaan. Eli siis muokkaamaan tiedostoa ilman sudoa.

Hakemistolle:

![new permissions](imagesh3/new_permissions.JPG)
```
-R, tekee oikeuden muutokset koko hakemisto polulle
```
```
ls -l, tulostaa listauksen pitkässä muodossa, jotta nähdään myös oikeudet.
```
Tiedostolle:

![index](imagesh3/index_permissions.JPG)

Kokeillaan pääseekö testi Pena testaamaan hakemistoon uusien tiedostojen tekemistä. Ei pääse!

![pena no](imagesh3/pena_cant_write.JPG)

Entäpä muokkaamaan? Kyllä pääsee!

![pena yes](imagesh3/pena_can_edit.JPG)

![pena_page](imagesh3/pena_page.JPG)

Go, Pena!

## b) Moottorix. Asenna Nginx käsin. 
> Weppisivun tulee näkyä palvelimen etusivulla. Sivun tulee olla tavallisen käyttäjän muokattavissa, ilman root- tai sudo-oikeuksia. (Muista sammuttaa Apache ensin.)

Apache kiinni komennolla:
```
sudo systemctl stop apache2
```
### Ladataan nginx.
Minulla jo löytyy nginx, mutta näin se menisi jos ei olisi:
```
sudo apt install nginx
```

![install nginx](imagesh3/install_nginx.JPG)
### Potkitaan se käyntiin
```
sudo systemctl start nginx
```
Sitten vakiosivut pitäisi löytyä "http://localhost"

![vakiosivu](imagesh3/apache2_defaultpage.JPG)

Vakiosivut näköjään ovat oletuksena samassa hakemistossa. Siksi varmaankin index.html tiedosto näkyy molemilla palvelimilla.

![nginx path](imagesh3/nginx_vakiopolku.JPG)

Jos annan uuden konfiguraation nginx:lle, ehkä se ratkeaa.

### Laitetaan uusi hakemisto nginx:lle ja kopioidaan sen vakiosivu sinne sekä muutetaan sille samat oikeudet kuin apachelle. Sitten konfiguraatiot.
```
mkdir /var/www/nginx
```

![dir](imagesh3/nginx_ownPath.JPG)
```
sudo cp /var/www/html/index.nginx-debian.html /var/www/nginx/
```
![cp](imagesh3/cp_nginx.JPG)

Sitten annetaan samat oikeudet kuin apachen vakiosivulle ja hakemistolle.

![permissions](imagesh3/nginx_permissions_path.JPG)

### NGINX KONFIGUROINTI
Nginx:n vakiosivun konfiguraatio tiedosto löytyy /etc/nginx/sites-available/default.

![conf](imagesh3/nginx_conf.JPG)

Muutetaanpas sitä, että hakemisto on oikein. Eli vaihdetaan "root  /var/www/html" lopusta "html" => "nginx".
"root  /var/www/nginx"

![new conf](imagesh3/nginx_conf2.JPG)

### Nginx uudelleen käynnistys, jotta muutokset tulevat voimaan. 
```
sudo systemctl restart nginx
```
![nginx vakiosivu](imagesh3/nginx_defaultpage.JPG)

Ja näin se onnistui!

## c) Automoottorix. Automatisoi Nginx asennus Ansiblella. 
> Ylläpitäjän osuus Ansiblella riittää, itse HTML-weppisivut voi tehdä käsin.

Tarvitsen roolin nginx jolle alahakemistot "files", "handlers" ja "tasks". Konfiguraatiotiedosto menee "files" hakemistoon, josta se kopioidaan kohdekoneille. "Handlers" hakemistoon menee tehtävä, joka käynnistää uudelleen nginx demonin jos konfiguraatio muuttuu. "Tasks" hakemistoon tulee tiedosto jolla ladataan nginx, siirretään konfiguraatiotiedosto kohdekoneelle ja asetetaan käyttöoikeudet.

![first fail](imagesh3/first_fail.JPG)

Ihme juttu, kaatuu ennen kuin yrittää. Ajetaan sitten playbook parametrillä "-K"(kysyy salasanaa sudoa varten). En oikein ymmärrä ongelmaa, mutta voisi kokeilla näin.

![second fail](imagesh3/second_fail.JPG)

Vähän lupaavampi! Ongelma nyt selkeästi omassa huolimattomuudessa. Sen siitä saa, kun ei tarkista tarkasti, kadut varmasti.
Käydäänpäs muuttamassa tuo "apache2" pois ja laitetaan tilalle "nginx".

![toimii](imagesh3/success.JPG)

Lopputulema on hyvä. Lataa nginx, kopioi konfiguraatiotiedosto ja uudelleen käynnistä nginx. Handler hoitaa uudelleen käynnistyksen ja default file määrää verkkosivun asetukset(polku, portit jne.).
