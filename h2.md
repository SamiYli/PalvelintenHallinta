### copy
Kopioi tiedostoja tai sisältöä kohdekoneelle.

Options:
- content: tiedoston sisältö suoraan playbookista  
- dest: kohdepolku  
- src: lähdetiedosto  
- owner: omistaja  
- group: ryhmä
- mode: kohdetiedoston käyttöoikeudet
Esimerkki:  
```
- name: Kopioi tiedosto palvelimelle
  copy:
    src: /local/path/tiedosto.txt
    dest: /remote/path/tiedosto.txt
```
### apt
Hallitsee paketteja Debian-pohjaisissa järjestelmissä.

Options:
- name: paketin nimi  
- state: haluttu tila (present, absent, latest)
- update_cache: päivittää pakettivaraston  

Esimerkki:
```
- name: Asenna nginx
  apt:
    name: nginx
    state: present
    update_cache: yes
```
### file  
Hallitsee tiedostojen ja hakemistojen ominaisuuksia.

Options:
- path: kohdepolku  
- recurse: rekursiivinen käsittely  
- src: lähde (esim. linkeissä)  
- state: tila (file, directory, link, absent)  
- owner: omistaja  
- group: ryhmä  
- mode: käyttöoikeudet  

Esimerkki:
```
- name: Luo hakemisto
  file:
    path: /opt/esimerkki
    state: directory
    mode: '0755'
```
### user
Hallitsee käyttäjätilejä kohdekoneella.

Options:
- name: käyttäjän nimi  
- create_home: luodaanko kotihakemisto  
- comment: kuvaus  
- groups: ryhmät  
- shell: oletus shell  
- state: present/absent  
- system: onko järjestelmäkäyttäjä  

Esimerkki:
```
- name: Luo käyttäjä
  user:
    name: antero
    state: present
    shell: /bin/bash
```
### authorized_key 
Lisää SSH-avaimia käyttäjän authorized_keys-tiedostoon.

Options:
- user: käyttäjä  
- key: julkinen avain  

Esimerkki:  
```
- name: Lisää SSH-avain käyttäjälle
  authorized_key:
    user: antero
    key: "ssh-ed25519 AAAAC3Nza... käyttäjä@kone"
```

## a) Sudoless. Tee ansiblea varten tunnus, jolla voi käyttää sudoa ilman salasanaa. Sekä ssh-kirjautuminen että sudon käyttö tulee olla ansbilea varten automatisoitu.

### Käyttäjien lisääminen vaatii superuser oikeudet. 
Lisään käyttäjän nimeltä anster seuraavalla komennolla:
```
sudo adduser anster
```
### Annetaan anster käyttäjälle sudo oikeudet lisäämällä se sudoers ryhmään:
```
sudo adduser anster sudo
```
### Nyt lisätään ryhmä jolle annetaan salasanaton sudo oikeus komennolla:
```
sudo groupadd sudoless
```
### Lisätään käyttäjä anster ryhmään adm ja sudoless:
```
sudo adduser anster adm
sudo adduser anster sudoless
```
Anster kuuluu adm ryhmään, näin tehtiin tunnilla (2.4.2026), tämä oli muistinvarainen teko. Miksiköhän näin tehdään?

![]()

### Lisätään sudoers.d määritystiedostoon oikeus salasanattomaan toimintaan ryhmälle sudoless:
```
%sudoless ALL=(ALL) NOPASSWD:ALL
```
```
% tarkoittaa ryhmää, sudoless on ryhmän nimi, ALL tarkoittaa kaikkia käyttäjiä,
(ALL) sallii kaikki toiminnot, NOPASSWD:ALL poistaa kaikilta ryhmän jäseniltä salasana kyselyn.
```

![]()

## (Lisäyksiä 8.4.2026)
Nyt minulla on käyttäjä anster, jolla on oikeudet käyttää sudoa ilman salasanaa. Anster kuuluu ryhmään sudoless. Jos haluan lisää samat oikeudet omaavia käyttäjiä, voin lisätä ne sudoless ryhmään. Aika kiva, mutta Anster joutuu vieläkin näppäillä salasanan kirjautuessaan ssh yhteyden kautta. Annan siis seuraavaksi ansterille SSH avaimen, jottei tarvitse enää nysvätä salasanoja.
```
ssh-copy-id anster@localhost  // tallentaa julkisen avaimen
```

![]()

Tätähän voisi testata ottamalla ssh yhteyden localhost:iin
```
ssh anster@localhost
```
Ja tämähän toimii.

## b) Antero. Tee salasanaton, automaattisesti ssh:lla kirjautuva tunnus Ansiblella. (Lisäyksiä 8.4.2026)

Minulla on h1 tehtävän jäljiltä ansiblea varten työkansio ja toimiva hello world. Sen viritelmän voi käydä katsomassa [täältä]().
Tässä vaiheessa tein samankaltaisen viritelmän. Roolit hakemistoon lisäsin alihakemiston "anster" ja sen alle "tasks" hakemisto, jonka alle main.yml tiedosto. Main.yml tiedostoon kirjoittelen tehtävät, joita ansible suorittaa. Sen sisältö on sama kuin Tero Karvisen esimerkki, mutta ssh avain on oma.
[Tero Karvinen, Passwordless Sudo with Ansible, Role - antero](https://terokarvinen.com/passwordless-sudo-with-ansible/):
```
- group:                       // Luo ryhmän sudoless
    name: "sudoless"
    state: present
- user:                        // Luo käyttäjän "antero"
    name: "antero"
    state: present
    groups: ["sudoless", "sudo", "adm"]     // Lisää käyttäjän näihin ryhmiin.
- authorized_key:                           // Antaa ssh avaimen käytettäväksi
    user: "antero"
    key: "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIGKCn3ohP5nLEvv7XjRfYAvbgqAKvOuUUbT7pmwJr60z sami@vbox"       
- copy:                                  // Kopioi määritystiedostoon ohjeen salasanattomaan toimintaan ryhmälle sudoless 
    dest: "/etc/sudoers.d/sudoless"
    content: "%sudoless ALL = (ALL) NOPASSWD: ALL\n"
    owner: "root"
    group: "root"
    mode: "0644"
```
Innostuin tekemään vielä erillisen playbook tiedoston "testaa_sudo.yml", joka kertoo mitkä koneet omaksuvat roolin anster ja suorittavat sen tehtävät.

```
- hosts: all
  become: true
  roles:
    - anster
```
Sitten suorittamaan playbook komennolla:
```
ansible-playbook -K testaa_sudo.yml
```
