## a) Sshecrets.
Asenna SSH-demoni ja testaa se kirjautumalla SSH:lla.
OpenSSH aika hyvä tässä. Minulla on se jo ladattuna, mutta sen voi ladata terminaalissa komennolla:
```
sudo apt-get install openssh-server
```
Sitten sitä voidaan ajaa komennolla:
```
sudo systemctl start ssh
```
Yhdistäminen paikallisesti onnistuu komennolla:
```
ssh user@localhost
```
Täytyy korvata "user" omalla käyttäjätunnuksella, muuten ei toimi.

![openssh_pswd]( )

Onpas tympeetä kirjoitella salasanoja. Lisäämällä julkisen avaimen tälle palvelimelle, voisin ehkä säästää vaivaa ja aikaa.

## b) Pubkey.
Automatisoi ssh-kirjautuminen julkisella avaimella.
Keygen työkalulla voi luoda uuden avainparin. Se luo heti kivasti salaisen avaimen ja jemmaa sen piiloon.
Sitten halutessaan voi antaa salasanattoman pääsyn lähettämällä julkisen avaimen palvelimelle käytettäväksi.

```
ssh-copy-id user@palvelin
```
Tämä siirtää julkisen avaimen tiedostoon:
```
~/.ssh/authorized_keys
```
Luodaan uusi avain:
```
ssh-keygen
```
![keygen]()

Nyt meillä on uusi avainpari, annetaanpas se ssh-palvelimelle käytettäväksi.
```
ssh-copy-id kali@localhost
```
![copy-id]()


## c) Hei Ansible.
Tee hei maailma ansiblella ja kokeile sitä SSH:n yli.

Tein ensiksi hakemiston jonne värkkäillä. Sitten lähdin tekemään inventaario tiedostoa palvelimista, käyttämällä micro editoria. (Micro ei ollut ladattuna, mutta sain promptina ladataanko N/Y?)

![dir ansible]()

![hosts ini]()
Nyt minulla on paikallinen ssh palvelin listalla.

Testaillaanpa löytyykö
```
ansible all -a 'uptime' i hosts.ini
```
Eikä komennolla tapahtunut mitään. Ei varmasti tapahdu, kun parametrit on kirjoitettu väärin. !! [-i] !!
Kokeillaanpa uudelleen:

![ansible time]()

Nyt toimii!

Lisätään hosts.ini ansible.cfg tiedostoon, ettei tarvitse jatkossa kirjoitella -i hosts.ini.
```
micro ansible.cfg
inventory = hosts.ini
```
![ansible cfg]()

!!!!! TAAS PIKKU VIRHE !!!!!

Olisi pitänyt kirjoittaa .cfg tiedostoon:
```
[defaults]
inventory = hosts.ini
```

Mutta nyt toimii:

![ansible cfg2]()

Tehdään playbook tähän väliin. Se on siis tiedosto jossa määritellään mitä tehdään ja mille koneille. [Ansible playbook](https://docs.ansible.com/projects/ansible/latest/playbook_guide/playbooks_intro.html)
Micro hello.yml
```
- hosts: all
  roles:
    - hello
```
Nyt tuolta ansible siis lukee, että kaikki palvelimet suorittaa roolin "hello". 

![hello playbook]()

Sitten tehdään roolit:
```
mkdir -p roles/hello/tasks/
```
Sitten kirjoitellaan tähän polkuun ansible/roles/hello/tasks/ tiedosto main.yml
```
- copy:
    dest: /tmp/hello-ansible
    content: "Näkyykö tää sulle?\n"
```
![main.yml]()
